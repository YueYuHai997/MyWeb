## 迁移动机

原页面把首屏列表、标签统计和推荐模块都放在客户端请求，导致首屏完整内容依赖多个接口返回。内容站这种读多写少的场景，没必要把展示层全部压给浏览器。

## 迁移过程

我先把纯展示组件迁到 Server Component，只把筛选器和收藏按钮这类强交互模块留在 Client Component。关键不是全量替换，而是先划清哪些组件真的依赖浏览器状态。

## 结果

LCP 有明显改善，首屏可交互时间也更稳定。但缓存边界和组件职责要比以前严格，不然很容易出现一处变动牵出整页刷新。
